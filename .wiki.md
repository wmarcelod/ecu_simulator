# Project Summary
O projeto é um simulador de Unidade de Controle do Motor (ECU) baseado na web, que emula ECUs automotivas para desenvolvimento, teste e fins educacionais. Ele permite a interação em tempo real com dados de veículos, utilizando o protocolo ELM327 e comandos OBD-II. O simulador é uma ferramenta acessível e econômica que melhora os processos de engenharia automotiva e agora incorpora um modelo de aprendizado de máquina (ML) para gerar dados de sensores mais coerentes, além de suportar a simulação de ataques cibernéticos. A aba "RESEARCH" foi expandida para incluir relatórios de pesquisa da Iris e o design do simulador ECU web, oferecendo uma visão abrangente sobre capacidades de simulação e arquitetura do sistema.

# Project Module Description
O projeto consiste em vários módulos funcionais:
- **Dashboard Principal**: Painel de controle para simulação com seleção de modelo de veículo, exibição de dados em tempo real, controles de simulação e suporte para carregar arquivos DBC.
- **Motor de Simulação ECU**: Emula o protocolo ELM327 e responde a comandos OBD-II, agora com lógica de coerência entre sensores através de ML.
- **Perfis Multi-Modelo**: Perfis de veículos pré-configurados com PIDs e DTCs específicos.
- **Web Serial API**: Integração opcional para conexão com hardware Arduino/MCP2515.
- **Terminal ELM327**: Interface de comando interativa para enviar e receber comandos ELM327, com funcionalidade de exportação de logs.
- **Simulação Dinâmica de Sensores**: Ajuste de valores de sensores virtuais com gráficos em tempo real, agora com controle independente para simulação de ataques.
- **Playback Histórico**: Funcionalidade para upload e reprodução de logs de dados históricos da ECU.
- **Gerenciamento de DTCs**: Gestão de códigos de falha de diagnóstico, incluindo adição, remoção e limpeza de DTCs.
- **Schematic Panel**: Diagrama visual mostrando a conexão entre Arduino e MCP2515, com tabela de pinagem detalhada.
- **Painel de Pesquisa (RESEARCH)**: Nova aba que exibe documentos técnicos, incluindo relatórios de análise CAN bus, arquitetura do sistema, pesquisa da Iris e design do simulador ECU.
- **Documentação em LaTeX**: Descrição detalhada do simulador, abordando aspectos técnicos, funcionalidades e arquitetura.

# File Description Inventory
- **src/lib/ecu-simulator.ts**: Contém a lógica central para simular o comportamento da ECU, agora com integração de um modelo de ML para coerência entre sensores.
- **src/lib/serial-connection.ts**: Gerencia interações com a Web Serial API para conectar ao hardware.
- **src/lib/dbc-parser.ts**: Implementa o parser para arquivos DBC, permitindo a importação de definições de veículos.
- **src/components/Dashboard.tsx**: Exibe dados de veículos em tempo real através de medidores e controles, incluindo um botão para carregar arquivos DBC.
- **src/components/Terminal.tsx**: Fornece uma interface para enviar comandos ao simulador de ECU e exportar logs de comandos.
- **src/components/SensorPanel.tsx**: Permite ajuste manual de valores de sensores virtuais e exibe gráficos em tempo real.
- **src/components/PlaybackPanel.tsx**: Gerencia a reprodução de logs de dados históricos carregados pelo usuário.
- **src/components/DTCPanel.tsx**: Gerencia a exibição e interação com códigos de falha de diagnóstico.
- **src/components/SchematicPanel.tsx**: Exibe o diagrama de conexão entre Arduino e MCP2515, incluindo uma tabela de pinagem e notas técnicas.
- **src/components/ResearchPanel.tsx**: Painel atualizado que renderiza documentos de pesquisa, incluindo o relatório de pesquisa da Iris e o design do simulador ECU.
- **src/pages/Index.tsx**: Layout principal da aplicação que integra todos os componentes da UI, agora com aba de pesquisa.

# Technology Stack
- **Frontend**: React, TypeScript, Tailwind CSS
- **Backend**: Nenhum (lógica do lado do cliente apenas)
- **Hardware**: Arduino, MCP2515 controlador CAN (opcional)
- **Comunicação**: Web Serial API, protocolo OBD-II
- **Machine Learning**: Python, scikit-learn

# Usage
Para configurar o projeto:
1. **Instalar Dependências**: Utilize o gerenciador de pacotes para instalar as bibliotecas necessárias.
2. **Construir o Projeto**: Compile o projeto usando o comando de build.
3. **Executar a Aplicação**: Inicie a aplicação web para começar a usar o simulador no modo standalone, sem a necessidade de hardware.
